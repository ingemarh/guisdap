The following document describes how to use the ONERA-DESP fortran library to compute magnetic coordinates and drift shells using various external magnetic field models.  Further routines are provided for  various coordinate and time format transformations.

The library can be called from FORTRAN or C code and from IDL or MATLAB code. For IDL and MATLAB wrappers are provided in the distribution package. 

The package includes sripts to compile the library under linux, sun solaris and windows/cygwin systems. You might have to change it to build the library on your system. Currently the library is compiled using the standard gnu compiler (g77). Just know that the use of any other compiler like intel/fortran on linux can increase the speed by a factor of 2 to 3. An example calling the library from IDL is given in example directory of the distribution and can be run to test if the library has been built properly. Since version 4.1 a Makefile if provided to built the library on various platform, install it and test it (See section installation for more details). 

Two libraries are created, one for access from IDL or MATLAB and one for access from fortran or some other language.
     -from IDL or MATLAB use lib_name='onera_desp_lib.so' or 'onera_desp_lib.DLL' depending on your operating system.
     -from FORTRAN use liboneradesp.a

Notes:

1. The bad value flag is set to  -1.d31 throughout.

Caution: The library has so far been extensively tested with Olson Pfitzer quiet. Other magnetic field models should be used with caution, as there may be problems. Expect this message to evolve as we test more of the code.
